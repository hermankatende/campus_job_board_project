# Render Setup

This project is configured to deploy the Django backend on Render and use Supabase Postgres as the database.

## Render Service

- Service name: `campus-job-board-api`
- Environment: `Python`
- Root directory: `django_backend`
- Build command: `pip install -r requirements.txt && python manage.py collectstatic --noinput`
- Start command: `python manage.py bootstrap_deploy && gunicorn config.wsgi:application`

## Render Environment Variables

Set these in the Render dashboard for the web service.

```env
DJANGO_DEBUG=False
DJANGO_ALLOWED_HOSTS=campus-job-board-project.onrender.com
DJANGO_CSRF_TRUSTED_ORIGINS=https://campus-job-board-project.onrender.com
DB_ENGINE=django.db.backends.postgresql
DB_NAME=postgres
DB_USER=postgres.<your-project-ref>
DB_PASSWORD=your-actual-supabase-password
DB_HOST=aws-1-eu-west-1.pooler.supabase.com
DB_PORT=6543
DB_SSLMODE=require
```

Render can generate this automatically:

```env
DJANGO_SECRET_KEY=<generated by Render>
```

To create the deploy-time admin automatically, also set:

```env
DJANGO_SUPERUSER_USERNAME=herman
DJANGO_SUPERUSER_PASSWORD=12345herman
DJANGO_SUPERUSER_EMAIL=herman@example.com
```

Each deploy runs migrations and creates or updates that superuser automatically.

Firebase configuration still needs one of these:

```env
FIREBASE_CREDENTIALS_JSON={...your Firebase service account JSON...}
```

or

```env
FIREBASE_CREDENTIALS_FILE=service-account.json
```

Optional email settings for lecturer verification:

```env
LECTURER_HOD_EMAIL=hod@example.com
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password
EMAIL_USE_TLS=True
DEFAULT_FROM_EMAIL=your-email@gmail.com
```

## Flutter App

The app now points to Render through the root `.env` file:

```env
BACKEND_URL=https://campus-job-board-project.onrender.com
```

You can also override it when building:

```bash
flutter run --dart-define=BACKEND_URL=https://campus-job-board-project.onrender.com
```

## Important

- Rotate the Supabase password before using it in Render.
- If Render assigns a different service URL, update `DJANGO_ALLOWED_HOSTS`, `DJANGO_CSRF_TRUSTED_ORIGINS`, and `BACKEND_URL` to match it.
